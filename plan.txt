# 金融风控工具平台 (FinRiskTool) 项目分析报告

## 1. 项目概述
本项目是一个基于 Python Flask 框架构建的综合性金融风控工具平台。它融合了传统的机器学习方法（PU Learning、集成特征选择）和前沿的大语言模型技术（Chain of Thought 推理、自动 Prompt 优化），旨在提供从数据处理、模型训练到风险分析推理及结果质检的全流程解决方案。

## 2. 系统架构

### 2.1 技术栈
- **Web 框架**: Flask
- **数据处理**: Pandas, NumPy, Scikit-learn
- **机器学习**: LightGBM, XGBoost
- **大模型集成**: OpenAI SDK (兼容 DeepSeek 等模型)
- **并发处理**: Python Threading, ThreadPoolExecutor
- **前端**: HTML模板 (Jinja2)

### 2.2 目录结构与模块划分
项目采用模块化设计，主要分为路由层 (`routes`) 和服务层 (`services`)。

*   **app/routes/**: 处理 HTTP 请求，定义 API 接口和页面路由。
    *   `data_tool.py`: 传统风控数据工具路由。
    *   `risk_cot/`: LLM 风险思维链相关路由。
*   **app/services/**: 核心业务逻辑实现。
    *   `data_core/`: 传统机器学习算法实现。
    *   `risk_cot/`: LLM 推理与质检引擎实现。
## 3. 核心模块详解

### 3.1 数据工具模块 (Data Tool)
该模块主要处理结构化表格数据，提供传统的风控建模能力。

*   **PU Learning (PU_bagging.py)**
    *   **功能**: 解决正样本（违约）稀缺、未标记样本大量的场景。
    *   **实现**: 采用 Bagging 策略，多次迭代从未标记样本中采样负样本，训练多个 LightGBM 基分类器并集成。
    *   **输出**: 违约风险概率预测、特征重要性。

*   **集成特征选择 (ensemble_feature_selection.py)**
    *   **功能**: 筛选高价值风控特征。
    *   **实现**: 结合互信息 (Mutual Information)、XGBoost、RandomForest 三种方法的特征重要性，加权投票 (0.3/0.4/0.3)。
    *   **策略**: 对比三种数据集（原始、高置信度负转正、伪正样本补充）的特征表现。

### 3.2 风险思维链模块 (Risk CoT)
该模块利用 LLM 的推理能力进行深度风险分析。

*   **数据工厂 (RiskDataFactory)**
    *   **功能**: 生成包含企业经营、财务、贷后检查文本的模拟数据。
    *   **特点**: 模拟真实业务场景，生成异构数据（数值+文本）。

*   **Prompt 引擎 (PromptEngine)**
    *   **功能**: 自动化 Prompt 工程。
    *   **核心能力**:
        *   **模板生成**: 调用 LLM 根据特征列表自动优化 Prompt 模板。
        *   **数据转换**: 将表格数据填充进模板，生成 Alpaca 格式 (`instruction`, `input`, `gt`) 数据集。

*   **推理引擎 (InferenceEngine)**
    *   **功能**: 批量执行 LLM 推理任务。
    *   **特性**:
        *   支持 DeepSeek R1 等推理模型。
        *   多线程并发加速。
        *   支持断点续传（基于内容 Hash）。
        *   解析 `<think>` 思维链和 `<answer>` 结果。

*   **质检引擎 (InspectorEngine)**
    *   **功能**: 对 LLM 推理结果进行质量评估。
    *   **双重质检**:
        1.  **规则质检 (RuleBase)**: 基于正则表达式检查格式、截断、重复、敏感词、数学一致性等 10+ 项指标。
        2.  **模型质检 (ModelScorer)**: 使用 LLM 作为裁判，从准确性、思维链难度、合成质量三个维度打分。

## 4. 文件功能清单

### 4.1 应用层 (app/)
| 文件路径 | 功能描述 |
| :--- | :--- |
| `__init__.py` | Flask 应用工厂，配置日志、目录及注册蓝图。 |
| `routes/main.py` | 首页路由。 |
| `routes/data_tool.py` | 数据工具 API，处理文件上传及调用 Data Core 脚本。 |
| `routes/risk_cot/generator.py` | 数据生成与 Prompt 模板 API。 |
| `routes/risk_cot/inference.py` | 推理任务管理 API (启动、停止、状态查询)。 |
| `routes/risk_cot/inspector.py` | 质检任务管理 API。 |
| `routes/risk_cot/views.py` | Risk CoT 前端页面路由。 |

### 4.2 服务层 (app/services/)
| 文件路径 | 功能描述 |
| :--- | :--- |
| **Data Core** | |
| `data_core/PU_bagging.py` | PU Learning 算法实现 (Bagging + LightGBM)。 |
| `data_core/ensemble_feature_selection.py` | 集成特征选择算法 (MI + XGB + RF)。 |
| `data_core/generate_data.py` | 简单的随机数据生成脚本。 |
| `data_core/split_data.py` | 数据集切分工具。 |
| `data_core/MLBaseModel.py` | 基础模型训练实验代码 (GridSearch, 阈值调优)。 |
| **Risk CoT** | |
| `risk_cot/data_factory.py` | 复杂的风控模拟数据生成器。 |
| `risk_cot/prompt_engine.py` | Prompt 模板管理与数据格式化。 |
| `risk_cot/inference_engine.py` | 多线程 LLM 推理执行引擎。 |
| `risk_cot/inspector_engine.py` | 质检任务调度引擎。 |
| `risk_cot/model_inspector.py` | 基于 LLM 的模型打分逻辑。 |
| `risk_cot/rule_base.py` | 基于规则的详细质量检查逻辑。 |
